잡헌터(구인구직 플랫폼)

6인 팀 프로젝트

잡헌터는 사용자 친화적 인터페이스를 기반으로 한 구직자와 기업을 연결하는 구인구직 플랫폼입니다. Spring 프레임워크를 활용하여 개발되었으며, 저는 1:1채팅 상담 기능과 고객센터 게시판을 구현했습니다. 초기에는 일반 컨트롤러를 사용하여 모든 페이지를 구현하려 했으나, 실시간 상호작용의 필요성을 인식하고 아키텍처를 개선했습니다. 1:1 채팅 기능은 REST API와 비동기 폴링 기법을 결합한 접근 방식을 채택했습니다. 서버 측에서는 채팅 메시지를 저장하고 조회하는 엔드포인트를 구현했고, 클라이언트 측에서는 fetch API를 사용하여 주기적으로 새 메시지를 폴링하는 방식으로 실시간 채팅과 유사한 효과를 제공했습니다. 특히, setInterval 함수를 활용하여 5초마다 서버로부터 최신 채팅 목록을 가져오는 reloadChat 함수를 구현했습니다. 이를 통해 실시간성을 확보했습니다. 또한, 동적 UI 렌더링에 중점을 두어 데이터의 상태와 사용자 권한에 따라 사용자 맞춤형 정보를 제공하여 UX를 크게 향상시켰습니다.

잡빙수(구인구직 플랫폼)

5인 팀 프로젝트

잡빙수는 신선한 UI/UX 디자인과 매칭 알고리즘을 제공하는 구인구직 플랫폼입니다. Spring 프레임워크를 활용하여 개발되었으며, 저는 기업 페이지 관리, 기업 공고 관리, 지원자 관리와 같은 기업과 관련된 핵심 기능을 구현했습니다. 초기에는 하나의 서버만을 사용하여 모든 기능을 구현하려 했으나, 효율성과 확장성을 위해 뷰 서버와 REST 서버를 분리하여 크로스 오리진으로 정보 공유를 구현했고, 데이터와 관련된 모든 주요 기능을 REST API로 처리하는 방식을 채택했습니다. 그 결과 유지보수성과 확장성을 높였습니다. 서버 측에서는 JPA를 활용하여 데이터 처리 계층을 구축했고, 클라이언트 측에서는 최신 JavaScript 기술을 활용하여 동적인 UI를 구현했습니다. 자체 매칭 알고리즘과 인공지능 API를 이용한 알고리즘을 비교 분석하여 매칭 알고리즘의 성능을 향상시켜 서비스의 품질과 다양성을 높였습니다.

뭐뭐뭐(종합 요리 플랫폼)

6인 팀 프로젝트

뭐뭐뭐는 사용자 맞춤형 레시피 추천과 날씨 기반 메뉴 추천을 제공하는 종합 요리 플랫폼입니다. Spring Boot 프레임워크를 활용하여 개발되었으며, 저는 메뉴 및 레시피 관련 페이지, 날씨 기반 메뉴 추천 시스템과 같은 기능을 구현했습니다. 서버 측에서는 MyBatis를 활용하여 복잡한 데이터 쿼리를 효율적으로 처리했으며, 클라이언트 측에서는 JavaScript 기술을 활용하여 동적이고 반응성 높은 UI를 구현했습니다. 사용자가 레시피 카테고리, 테마, 제외할 재료 등을 동시에 선택할 수 있게 하고, 이러한 복합 조건을 하나의 쿼리로 처리하여 정확하고 빠른 검색 결과를 제공했습니다. 또한, 검색어를 통해 레시피 이름, 내용, 카테고리, 테마를 동시에 검색할 수 있는 통합 검색 기능을 구현하여 사용자 경험을 향상시켰습니다. 마지막으로, OpenWeatherMap API를 활용한 날씨 기반 메뉴 추천 시스템을 구현하고 날씨 데이터와 메뉴 데이터를 결합하는 알고리즘을 개발하여 상황에 맞는 메뉴를 추천함으로써 서비스의 실용성을 높였습니다.
